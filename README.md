# E-Commerce Database Architecture Case Study

## Project Overview
This repository contains a comprehensive, production-grade relational database design and data modeling case study for a live national e-commerce platform (**CanadaFlowers.ca**). 

The goal of this project was to reverse-engineer complex e-commerce business operations—spanning user authentication, dynamic product catalog pricing, multi-recipient transactional logistics, and promotional token frameworks—into a scalable, high-integrity relational schema.

## Core Technical Skills Demonstrated
* **Requirements Analysis:** Translating real-world, full-stack business workflows into structured technical data elements.
* **Data Modeling:** Designing robust Entity-Relationship Diagrams (ERDs) handling strong/weak entities and complex many-to-many relationship abstractions.
* **Database Optimization & Normalization:** Mapping functional dependencies and normalizing data layouts up to **Third Normal Form (3NF)** to completely eliminate insertion, update, and deletion anomalies.
